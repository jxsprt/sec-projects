Encrypted File Vault

Build a personal file vault that encrypts/decrypts files using AES-256 with a password-derived key.

python aes cryptography

Learning Objectives

  • Understand symmetric encryption: how AES-256-GCM provides both confidentiality and authenticity
  • Learn key derivation: turning a human password into a strong cryptographic key using PBKDF2
  • Grasp the role of salt, nonce (IV), and authentication tags in preventing replay and tampering attacks
  • Design a file format that stores encrypted data + metadata (salt, nonce, tag) for reliable decryption
  • Understand why ECB mode is dangerous and GCM is the modern standard

Prerequisites

  • Python 3.8+ installed
  • pip install cryptography (the cryptography library, not the built-in module)
  • Basic understanding of what encryption does (plaintext → ciphertext)
  • No prior cryptography knowledge required — this project teaches it
⚠ Important Security Note

This project is for educational purposes. Do not roll your own crypto for production systems. The cryptography library used here exposes well-audited primitives correctly, but file format design and key management still require careful review before real-world use.

Step 1: Install Dependencies & Understand the Crypto Stack

Install the cryptography library — this gives us AES-GCM and PBKDF2 with safe defaults.

pip install cryptography

# Verify installation
python -c "from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes; print('Ready')"

We'll use three cryptographic primitives:

  • PBKDF2-HMAC-SHA256 — turns a password into a 256-bit key (100,000 iterations)
  • AES-256-GCM — encrypts data AND produces an authentication tag
  • os.urandom — generates cryptographically secure random salt and nonce

Step 2: Key Derivation with PBKDF2

Never use a password directly as an encryption key. PBKDF2 applies many iterations of a hash function to make brute-force attacks computationally expensive.

import os
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2HMAC
from cryptography.hazmat.backends import default_backend

def derive_key(password: str, salt: bytes = None) -> tuple:
    """Derive a 256-bit AES key from a password using PBKDF2."""
    if salt is None:
        salt = os.urandom(16)  # 128-bit random salt

    kdf = PBKDF2HMAC(
        algorithm=hashes.SHA256(),
        length=32,              # 256-bit key for AES-256
        salt=salt,
        iterations=100_000,     # OWASP recommended minimum
        backend=default_backend()
    )
    key = kdf.derive(password.encode('utf-8'))
    return key, salt

Why 100,000 iterations? Each additional iteration doubles the work an attacker must do. At 100,000 iterations, a single password guess takes ~0.1s on modern hardware — trivially fast for legitimate use, devastating for brute-force attacks.

Step 3: AES-256-GCM Encryption & Decryption

GCM (Galois/Counter Mode) is an authenticated encryption mode. It encrypts the data AND produces an authentication tag. If anyone tampers with the ciphertext, decryption fails — you can trust the output.

import os
import struct
from cryptography.hazmat.primitives.ciphers import Cipher, algorithms, modes

MAGIC = b'EFV1'  # Encrypted File Vault v1 — format identifier

def encrypt_file(input_path: str, password: str) -> str:
    """Encrypt a file with AES-256-GCM. Returns path to .enc file."""
    key, salt = derive_key(password)
    nonce = os.urandom(12)  # 96-bit nonce for GCM

    # Read plaintext
    with open(input_path, 'rb') as f:
        plaintext = f.read()

    # Encrypt
    cipher = Cipher(algorithms.AES(key), modes.GCM(nonce),
                    backend=default_backend())
    encryptor = cipher.encryptor()
    ciphertext = encryptor.update(plaintext) + encryptor.finalize()
    tag = encryptor.tag  # 16-byte authentication tag

    # Write encrypted file with header
    output_path = input_path + '.enc'
    with open(output_path, 'wb') as f:
        f.write(MAGIC)                    # 4 bytes: format magic
        f.write(struct.pack('>I', 16))    # 4 bytes: salt length
        f.write(salt)                     # 16 bytes: salt
        f.write(struct.pack('>I', 12))    # 4 bytes: nonce length
        f.write(nonce)                    # 12 bytes: nonce
        f.write(struct.pack('>I', 16))    # 4 bytes: tag length
        f.write(tag)                      # 16 bytes: auth tag
        f.write(ciphertext)               # remainder: encrypted data

    print(f"Encrypted: {input_path} → {output_path}")
    return output_path

def decrypt_file(enc_path: str, password: str) -> str:
    """Decrypt an AES-256-GCM encrypted file. Returns path to decrypted file."""
    with open(enc_path, 'rb') as f:
        magic = f.read(4)
        if magic != MAGIC:
            raise ValueError("Not an Encrypted File Vault file (bad magic)")

        salt_len = struct.unpack('>I', f.read(4))[0]
        salt = f.read(salt_len)

        nonce_len = struct.unpack('>I', f.read(4))[0]
        nonce = f.read(nonce_len)

        tag_len = struct.unpack('>I', f.read(4))[0]
        tag = f.read(tag_len)

        ciphertext = f.read()

    # Re-derive key from password + stored salt
    key, _ = derive_key(password, salt)

    # Decrypt
    cipher = Cipher(algorithms.AES(key), modes.GCM(nonce, tag),
                    backend=default_backend())
    decryptor = cipher.decryptor()

    try:
        plaintext = decryptor.update(ciphertext) + decryptor.finalize()
    except Exception:
        raise ValueError("Decryption failed — wrong password or corrupted file")

    output_path = enc_path.replace('.enc', '.dec')
    with open(output_path, 'wb') as f:
        f.write(plaintext)

    print(f"Decrypted: {enc_path} → {output_path}")
    return output_path

File format design: The header stores salt, nonce, and tag with length prefixes. This is self-describing — you can decrypt without any side-channel metadata. The MAGIC bytes prevent accidentally feeding the wrong file type to the decryption function.

Step 4: Command-Line Interface

Wrap the crypto functions in a user-friendly CLI with secure password input.

import argparse
import getpass
import sys
from pathlib import Path

def main():
    parser = argparse.ArgumentParser(
        description="Encrypted File Vault — AES-256-GCM file encryption"
    )
    sub = parser.add_subparsers(dest="cmd", help="Commands")

    enc = sub.add_parser("encrypt", help="Encrypt a file")
    enc.add_argument("file", help="File to encrypt")

    dec = sub.add_parser("decrypt", help="Decrypt a file")
    dec.add_argument("file", help="File to decrypt (.enc)")

    args = parser.parse_args()

    if args.cmd == "encrypt":
        if not Path(args.file).exists():
            print(f"Error: '{args.file}' not found")
            sys.exit(1)

        pwd = getpass.getpass("Encryption password: ")
        confirm = getpass.getpass("Confirm password: ")
        if pwd != confirm:
            print("Passwords do not match")
            sys.exit(1)

        try:
            encrypt_file(args.file, pwd)
        except Exception as e:
            print(f"Encryption failed: {e}")

    elif args.cmd == "decrypt":
        if not Path(args.file).exists():
            print(f"Error: '{args.file}' not found")
            sys.exit(1)

        pwd = getpass.getpass("Decryption password: ")
        try:
            decrypt_file(args.file, pwd)
        except ValueError as e:
            print(f"Decryption failed: {e}")
            sys.exit(1)

    else:
        parser.print_help()

if __name__ == "__main__":
    main()

Testing Your Vault

Verify the full encrypt → decrypt → integrity cycle:

# Create a test file
echo "TOP SECRET: The launch code is 4-8-15-16-23-42" > secret.txt

# Encrypt
python vault.py encrypt secret.txt
# → secret.txt.enc created

# Check the encrypted file — should look like random bytes
xxd secret.txt.enc | head -5

# Decrypt
python vault.py decrypt secret.txt.enc
# → secret.txt.dec created

# Verify integrity
diff secret.txt secret.txt.dec
# (should produce no output — files are identical)
