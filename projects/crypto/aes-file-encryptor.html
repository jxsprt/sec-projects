AES-256 File Encryptor

Implement AES-256-GCM file encryption with proper key derivation, authentication, and secure key storage.

Learning Objectives

Step 1: Key Derivation with PBKDF2

Never use passwords directly as encryption keys — they're low-entropy. Instead, derive a strong 256-bit key from the password using PBKDF2 with a random salt and high iteration count.

import os
from cryptography.hazmat.primitives.kdf.pbkdf2 import PBKDF2HMAC
from cryptography.hazmat.primitives import hashes
from cryptography.hazmat.backends import default_backend

def derive_key(password: str, salt: bytes = None, iterations: int = 600_000) -> tuple:
    """Derive a 256-bit AES key from a password using PBKDF2."""
    if salt is None:
        salt = os.urandom(16)  # 128-bit random salt

    kdf = PBKDF2HMAC(
        algorithm=hashes.SHA256(),
        length=32,          # 256-bit key for AES-256
        salt=salt,
        iterations=iterations,
        backend=default_backend()
    )
    key = kdf.derive(password.encode('utf-8'))
    return key, salt

# Example usage
password = "correct-horse-battery-staple"
key, salt = derive_key(password)
print(f"Derived key: {key.hex()}")
print(f"Salt: {salt.hex()}")

Key concepts: PBKDF2 makes brute-force attacks expensive. Each derivation takes ~0.1s at 600K iterations — negligible for legitimate use, devastating for attackers. The salt ensures identical passwords produce different keys.

Step 2: Encrypt a File with AES-256-GCM

AES-GCM provides both confidentiality (encryption) and integrity (authentication). The authentication tag prevents tampering — if even one byte is modified, decryption will fail.

import os
from cryptography.hazmat.primitives.ciphers.aead import AESGCM

def encrypt_file(input_path: str, output_path: str, password: str) -> None:
    """Encrypt a file using AES-256-GCM."""

    # Derive key from password
    key, salt = derive_key(password)

    # Generate a random 96-bit nonce (IV) — critical: NEVER reuse a nonce with the same key!
    nonce = os.urandom(12)

    # Read plaintext
    with open(input_path, 'rb') as f:
        plaintext = f.read()

    # Encrypt
    aesgcm = AESGCM(key)
    # Include salt + nonce as associated data (authenticated but not encrypted)
    associated_data = salt + nonce
    ciphertext = aesgcm.encrypt(nonce, plaintext, associated_data)

    # Write: salt (16) + nonce (12) + ciphertext (includes 16-byte tag)
    with open(output_path, 'wb') as f:
        f.write(salt)       # 16 bytes
        f.write(nonce)      # 12 bytes
        f.write(ciphertext) # plaintext + 16-byte authentication tag

    print(f"Encrypted '{input_path}' -> '{output_path}'")
    print(f"  Original size: {len(plaintext)} bytes")
    print(f"  Encrypted size: {len(ciphertext) + 28} bytes (+ salt/nonce overhead)")

Critical detail: The nonce (number used once) must be unique per encryption with the same key. Reusing a nonce with GCM catastrophically breaks security — an attacker can recover the authentication key and forge messages. We generate a random 96-bit nonce; at 296 possible values, collisions are astronomically unlikely.

Step 3: Decrypt the File

Decryption reads the salt and nonce from the file header, re-derives the key, and attempts decryption. If the authentication tag doesn't match, the file was tampered with.

from cryptography.exceptions import InvalidTag

def decrypt_file(input_path: str, output_path: str, password: str) -> bool:
    """Decrypt a file encrypted with AES-256-GCM. Returns True on success."""

    with open(input_path, 'rb') as f:
        salt = f.read(16)    # First 16 bytes
        nonce = f.read(12)  # Next 12 bytes
        ciphertext = f.read()  # Rest (includes 16-byte tag appended by GCM)

    if len(salt) != 16 or len(nonce) != 12:
        print("Error: Invalid encrypted file format")
        return False

    # Re-derive the same key from the password + stored salt
    key, _ = derive_key(password, salt)

    # Decrypt
    aesgcm = AESGCM(key)
    associated_data = salt + nonce

    try:
        plaintext = aesgcm.decrypt(nonce, ciphertext, associated_data)
    except InvalidTag:
        print("Error: Decryption failed — wrong password or file was tampered with!")
        return False

    with open(output_path, 'wb') as f:
        f.write(plaintext)

    print(f"Decrypted '{input_path}' -> '{output_path}'")
    print(f"  Decrypted size: {len(plaintext)} bytes")
    return True

The InvalidTag exception is your integrity guarantee. If it fires, you know definitively that either the password is wrong or the ciphertext was modified — GCM makes no distinction between the two, which is a security feature (prevents oracle attacks).

Step 4: Build the CLI Interface

Wrap everything into a usable command-line tool with argparse. Support both encrypt and decrypt modes with clean error handling.

import argparse
import sys
import os

def main():
    parser = argparse.ArgumentParser(
        description="AES-256-GCM File Encryptor/Decryptor"
    )
    subparsers = parser.add_subparsers(dest="command", required=True)

    # Encrypt subcommand
    enc_parser = subparsers.add_parser("encrypt", help="Encrypt a file")
    enc_parser.add_argument("input", help="Input file to encrypt")
    enc_parser.add_argument("-o", "--output", help="Output file (default: input.enc)")
    enc_parser.add_argument("-p", "--password", help="Encryption password (will prompt if omitted)")

    # Decrypt subcommand
    dec_parser = subparsers.add_parser("decrypt", help="Decrypt a file")
    dec_parser.add_argument("input", help="Input file to decrypt")
    dec_parser.add_argument("-o", "--output", help="Output file (default: strips .enc)")
    dec_parser.add_argument("-p", "--password", help="Decryption password (will prompt if omitted)")

    args = parser.parse_args()

    # Secure password input
    if args.password:
        password = args.password
        print("Warning: Passing password on command line is insecure (visible in shell history)")
    else:
        import getpass
        password = getpass.getpass("Password: ")

    if args.command == "encrypt":
        output = args.output or args.input + ".enc"
        if os.path.exists(output):
            resp = input(f"'{output}' already exists. Overwrite? [y/N] ")
            if resp.lower() != 'y':
                sys.exit(0)
        encrypt_file(args.input, output, password)

    elif args.command == "decrypt":
        output = args.output or args.input.replace(".enc", "")
        success = decrypt_file(args.input, output, password)
        if not success:
            sys.exit(1)

if __name__ == "__main__":
    main()

Testing Your Encryptor

Verify your implementation with these tests:

  • Round-trip test: Encrypt a file, decrypt it, and verify the original matches with sha256sum
  • Wrong password test: Try decrypting with the wrong password — you should get an InvalidTag error
  • Tamper test: Flip a byte in the encrypted file and try to decrypt — should fail
  • Nonce uniqueness: Encrypt the same file twice — outputs should be completely different due to random nonces
  • Large file test: Try a 100MB+ file (consider streaming for production use)
# Test round-trip
echo "Super secret data" > test.txt
python aes_encryptor.py encrypt test.txt -o test.txt.enc
python aes_encryptor.py decrypt test.txt.enc -o test_decrypted.txt
diff test.txt test_decrypted.txt && echo "SUCCESS: Files match"

Extension Ideas

  • Streaming encryption: Handle multi-GB files by encrypting in 64KB chunks (GCM supports this with incremental nonce construction)
  • Directory encryption: Recursively encrypt entire folders, preserving directory structure in the encrypted output
  • Key file support: Allow using a key file (like SSH keys) instead of a password for automated/headless use
  • Argon2 upgrade: Replace PBKDF2 with Argon2id for memory-hard key derivation (resists GPU/ASIC attacks)
  • GUI wrapper: Build a simple Tkinter or web interface for non-technical users
