RSA Key Generator & Message Signer

Generate RSA key pairs, encrypt/decrypt messages, and create/verify digital signatures — understand PKI foundations.

python rsa asymmetric-crypto signatures

Learning Objectives

  • Understand the mathematics behind RSA: prime generation, modular arithmetic, and the totient function
  • Learn the difference between encryption (confidentiality) and signing (authenticity/integrity)
  • Implement proper RSA key generation with industry-standard padding (OAEP, PSS)
  • Save and load keys in PEM format — the same format used by SSH, TLS, and X.509 certificates
  • Understand why RSA is used for key exchange, not bulk encryption

Prerequisites

  • Python 3.8+ with cryptography library (pip install cryptography)
  • Understanding of public/private key concepts (public = share, private = guard)
  • Basic familiarity with hexadecimal and Base64 encoding
  • Optional but helpful: familiarity with modular arithmetic concepts

Step 1: Generate an RSA Key Pair

Generate a 2048-bit RSA key pair using the cryptography library. The public exponent is set to 65537 — the industry standard that provides a good balance of security and performance.

from cryptography.hazmat.primitives.asymmetric import rsa
from cryptography.hazmat.primitives import serialization
from cryptography.hazmat.backends import default_backend

def generate_rsa_keypair(key_size: int = 2048) -> tuple:
    """Generate an RSA private/public key pair.

    Args:
        key_size: Key size in bits. 2048 minimum for security.
                  Use 4096 for long-term secrets.

    Returns:
        (private_key, public_key) tuple
    """
    private_key = rsa.generate_private_key(
        public_exponent=65537,       # Industry standard exponent
        key_size=key_size,
        backend=default_backend()
    )
    public_key = private_key.public_key()
    return private_key, public_key

# Generate keys
private_key, public_key = generate_rsa_keypair(2048)
print(f"Generated {private_key.key_size}-bit RSA key pair")
print(f"Public key: {public_key.key_size} bits")

Key concept: The public exponent 65537 (0x10001) is prime, has only two '1' bits, making modular exponentiation fast while still being large enough to resist low-exponent attacks like Coppersmith's attack.

Step 2: Encrypt and Decrypt Messages

RSA encrypts with the public key and decrypts with the private key. OAEP padding (Optimal Asymmetric Encryption Padding) is essential — never use raw RSA (aka "textbook RSA") as it's vulnerable to multiple attacks.

from cryptography.hazmat.primitives.asymmetric import padding
from cryptography.hazmat.primitives import hashes

def rsa_encrypt(public_key, plaintext: bytes) -> bytes:
    """Encrypt data with RSA-OAEP (SHA-256)."""
    ciphertext = public_key.encrypt(
        plaintext,
        padding.OAEP(
            mgf=padding.MGF1(algorithm=hashes.SHA256()),
            algorithm=hashes.SHA256(),
            label=None
        )
    )
    return ciphertext

def rsa_decrypt(private_key, ciphertext: bytes) -> bytes:
    """Decrypt data with RSA-OAEP (SHA-256)."""
    plaintext = private_key.decrypt(
        ciphertext,
        padding.OAEP(
            mgf=padding.MGF1(algorithm=hashes.SHA256()),
            algorithm=hashes.SHA256(),
            label=None
        )
    )
    return plaintext

# Example
message = b"Meet me at the old bridge at midnight"
print(f"Original: {message}")

ciphertext = rsa_encrypt(public_key, message)
print(f"Encrypted ({len(ciphertext)} bytes): {ciphertext.hex()[:60]}...")

decrypted = rsa_decrypt(private_key, ciphertext)
print(f"Decrypted: {decrypted}")

assert decrypted == message, "Round-trip failed!"

Size limitation: RSA can only encrypt data up to (key_size/8 - 2*hash_size - 2) bytes. With 2048-bit keys and SHA-256, that's ~190 bytes. In practice, RSA is used to encrypt a symmetric key (like an AES key), which then encrypts the actual data — a hybrid cryptosystem.

Step 3: Digital Signatures

Signing is the inverse of encryption: sign with the private key (proving only you could have created it) and verify with the public key. Use PSS padding for signatures (not PKCS1v15 which is considered legacy).

from cryptography.hazmat.primitives.asymmetric import padding, utils
from cryptography.exceptions import InvalidSignature

def sign_message(private_key, message: bytes) -> bytes:
    """Create a digital signature for a message."""
    signature = private_key.sign(
        message,
        padding.PSS(
            mgf=padding.MGF1(hashes.SHA256()),
            salt_length=padding.PSS.MAX_LENGTH
        ),
        hashes.SHA256()
    )
    return signature

def verify_signature(public_key, message: bytes, signature: bytes) -> bool:
    """Verify a digital signature. Returns True if valid."""
    try:
        public_key.verify(
            signature,
            message,
            padding.PSS(
                mgf=padding.MGF1(hashes.SHA256()),
                salt_length=padding.PSS.MAX_LENGTH
            ),
            hashes.SHA256()
        )
        return True
    except InvalidSignature:
        return False

# Example
message = b"Transfer $10,000 to account 987654321"
signature = sign_message(private_key, message)
print(f"Signature ({len(signature)} bytes): {signature.hex()[:60]}...")

# Verify
is_valid = verify_signature(public_key, message, signature)
print(f"Signature valid: {is_valid}")

# Tamper test — modifying the message invalidates the signature
tampered = message.replace(b"$10,000", b"$100,000")
is_valid = verify_signature(public_key, tampered, signature)
print(f"Tampered message valid: {is_valid}")  # False!

Critical distinction: Encryption provides confidentiality (only the private key holder can read). Signatures provide authenticity and integrity (proves who sent it and that it wasn't modified). Most real-world protocols need both.

Step 4: Key Serialization (Save & Load PEM Files)

Persist keys in PEM format — the universal standard used by SSH, TLS certificates, and virtually every crypto tool.

def save_private_key(private_key, filename: str, password: str = None):
    """Save private key to PEM file, optionally encrypted with a passphrase."""
    encryption = (
        serialization.BestAvailableEncryption(password.encode('utf-8'))
        if password
        else serialization.NoEncryption()
    )

    pem_data = private_key.private_bytes(
        encoding=serialization.Encoding.PEM,
        format=serialization.PrivateFormat.PKCS8,
        encryption_algorithm=encryption
    )

    with open(filename, 'wb') as f:
        f.write(pem_data)
    print(f"Private key saved to '{filename}'")

def save_public_key(public_key, filename: str):
    """Save public key to PEM file."""
    pem_data = public_key.public_bytes(
        encoding=serialization.Encoding.PEM,
        format=serialization.PublicFormat.SubjectPublicKeyInfo
    )

    with open(filename, 'wb') as f:
        f.write(pem_data)
    print(f"Public key saved to '{filename}'")

def load_private_key(filename: str, password: str = None):
    """Load a private key from PEM file."""
    with open(filename, 'rb') as f:
        pem_data = f.read()

    private_key = serialization.load_pem_private_key(
        pem_data,
        password=password.encode('utf-8') if password else None,
        backend=default_backend()
    )
    return private_key

def load_public_key(filename: str):
    """Load a public key from PEM file."""
    with open(filename, 'rb') as f:
        pem_data = f.read()

    public_key = serialization.load_pem_public_key(
        pem_data,
        backend=default_backend()
    )
    return public_key

# Usage
save_private_key(private_key, "my_private_key.pem", password="strong-passphrase")
save_public_key(public_key, "my_public_key.pem")

# Reload and verify
loaded_private = load_private_key("my_private_key.pem", password="strong-passphrase")
loaded_public = load_public_key("my_public_key.pem")
print(f"Reloaded key size: {loaded_private.key_size} bits")

Step 5: Complete CLI Wrapper

Build a full command-line tool with subcommands for generating keys, encrypting, decrypting, signing, and verifying.

import argparse
import sys

def main():
    parser = argparse.ArgumentParser(
        description="RSA Key Generator, Encryption, and Digital Signatures"
    )
    subparsers = parser.add_subparsers(dest="command", required=True)

    # Generate
    gen = subparsers.add_parser("generate", help="Generate RSA key pair")
    gen.add_argument("-s", "--size", type=int, default=2048, choices=[2048, 3072, 4096],
                     help="Key size (default: 2048)")
    gen.add_argument("--private", default="private.pem", help="Private key output")
    gen.add_argument("--public", default="public.pem", help="Public key output")
    gen.add_argument("-p", "--password", help="Passphrase for private key")

    # Sign
    sign_p = subparsers.add_parser("sign", help="Sign a file")
    sign_p.add_argument("file", help="File to sign")
    sign_p.add_argument("-k", "--key", required=True, help="Private key PEM file")
    sign_p.add_argument("--sig", default="signature.sig", help="Signature output")

    # Verify
    verify_p = subparsers.add_parser("verify", help="Verify a signature")
    verify_p.add_argument("file", help="Original file")
    verify_p.add_argument("-k", "--key", required=True, help="Public key PEM file")
    verify_p.add_argument("-s", "--sig", required=True, help="Signature file")

    args = parser.parse_args()

    if args.command == "generate":
        priv, pub = generate_rsa_keypair(args.size)
        save_private_key(priv, args.private, args.password)
        save_public_key(pub, args.public)
        print(f"Generated {args.size}-bit RSA key pair")

    elif args.command == "sign":
        priv = load_private_key(args.key)
        with open(args.file, 'rb') as f:
            data = f.read()
        sig = sign_message(priv, data)
        with open(args.sig, 'wb') as f:
            f.write(sig)
        print(f"Signed '{args.file}' -> '{args.sig}'")

    elif args.command == "verify":
        pub = load_public_key(args.key)
        with open(args.file, 'rb') as f:
            data = f.read()
        with open(args.sig, 'rb') as f:
            sig = f.read()
        if verify_signature(pub, data, sig):
            print(f"✓ Signature VALID — '{args.file}' is authentic")
        else:
            print(f"✗ Signature INVALID — file may be tampered!")
            sys.exit(1)

if __name__ == "__main__":
    main()

Testing Your Implementation

Verify correctness with these tests:

  • Round-trip: Encrypt and decrypt random data of various sizes (1-190 bytes)
  • Cross-compatibility: Generate a key, save as PEM, load with openssl rsa -in private.pem -text -noout
  • Signature tamper test: Modify signed data by 1 bit — signature must fail
  • Wrong key test: Try verifying with a different public key — must fail
  • Key size benchmark: Time key generation at 2048, 3072, and 4096 bits to understand the computational cost

Extension Ideas

  • Hybrid encryption: Combine RSA + AES — encrypt a random AES key with RSA, then bulk data with AES
  • Certificate chain validator: Extend signature verification to validate X.509 certificate chains
  • Key fingerprinting: Generate SSH-style key fingerprints (SHA256:base64) for key identification
  • Multi-recipient encryption: Encrypt a single file for multiple recipients by encrypting the AES key with each public key
  • Web-of-trust: Build a simple key signing and trust network — sign other people's public keys
